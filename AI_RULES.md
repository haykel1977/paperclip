# Tech Stack

- You are building a React application with TypeScript for type safety
- Use React Router for navigation between pages
- All source code should be placed in the src folder
- Pages should be organized in src/pages/ directory
- Components should be organized in src/components/ directory
- The main page (default page) is src/pages/Index.tsx
- Always update the main page to include new components so they're visible
- Use shadcn/ui library for UI components (already installed)
- Use Tailwind CSS for styling with extensive use of Tailwind classes
- Use lucide-react package for icons

## Library Usage Rules

- Use prebuilt components from the shadcn/ui library after importing them
- Do not modify the shadcn/ui component files directly
- Create new components if you need to customize shadcn/ui components
- Use lucide-react for all icon needs
- All styling should be done with Tailwind CSS classes
- Maintain consistent component structure and naming conventions
- Prefer functional components with hooks over class components
- Use TypeScript interfaces for component props and state types
- Leverage React's built-in hooks (useState, useEffect, useContext, etc.) appropriately
- Follow React best practices for performance optimization
- Ensure accessibility compliance with proper semantic HTML and ARIA attributes
